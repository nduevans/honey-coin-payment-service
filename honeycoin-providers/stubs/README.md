# Stub Servers

Two mock payment provider servers for local development.

## Setup

```bash
cd stubs
npm install
```

## Running

### ProviderAlpha (webhook-style, port 4001)

```bash
WEBHOOK_URL=http://localhost:3000/<your-path> node provider-alpha.js
```

`WEBHOOK_URL` must be the full URL including path. It defaults to
`http://localhost:3000/webhooks/provider-alpha` if not set.

### ProviderBeta (polling-style, port 4002)

```bash
node provider-beta.js
```

---

## API Contracts

### ProviderAlpha — port 4001

#### Initiate charge

```
POST /charge
Content-Type: application/json

{
  "requestId": string,
  "amount": number,
  "phoneNumber": string,
  "currency": string
}
```

Response:
```json
{ "providerRef": string, "status": "pending" }
```

#### Webhook callback

After 2–3 seconds, ProviderAlpha POSTs to `WEBHOOK_URL`:

```json
{ "providerRef": string, "status": "successful" | "failed" }
```

Your service must expose an endpoint at `WEBHOOK_URL` to receive this callback and
update the charge status accordingly.

---

### ProviderBeta — port 4002

#### Initiate charge

```
POST /charge
Content-Type: application/json

{
  "requestId": string,
  "amount": number,
  "phoneNumber": string,
  "currency": string
}
```

Response:
```json
{ "providerRef": string, "status": "pending" }
```

#### Poll status

```
GET /status/:providerRef
```

Response:
```json
{ "providerRef": string, "status": "pending" | "successful" | "failed" }
```

Returns `pending` for the first 2 calls per `providerRef`, then `successful`.

---

## Failure Trigger

Send `phoneNumber: "+00000000000"` to either provider's `/charge` endpoint to trigger
a failed outcome:

- **ProviderAlpha**: webhook callback will have `status: "failed"`
- **ProviderBeta**: first `/status` poll will return `status: "failed"`

---

## Notes

- Both stubs log all requests to stdout.
- `providerRef` is generated by the provider and is distinct from your `requestId`.
  Store it at charge initiation — you'll need it to correlate webhook callbacks
  (ProviderAlpha) or to poll for status (ProviderBeta).
- Stub state is in-memory and resets on restart. Your service's state must not.
